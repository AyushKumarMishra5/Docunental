# DocuIntel — Build Plan

A production-grade, enterprise document-intelligence platform. Next.js 14 (App Router) + TypeScript, Tailwind + customized shadcn/ui, OpenRouter (OpenAI-compatible), MongoDB Atlas, R2/local storage. Per your decisions: **swappable adapters with fixture-backed mock fallback** (every stage verifiable now, goes live with `.env.local` keys, zero code changes), **Graphite + Ink Blue** token system, and **all 8 stages built through** in order.

## 0. Foundations & design tokens

**Scaffold** (pnpm): `create-next-app` (TS, Tailwind, App Router, src/), then shadcn/ui initialized with the Graphite + Ink Blue theme (manual `components.json` + CSS vars so it's non-interactive). Deps: `framer-motion`, `recharts`, `pdf-parse`, `mammoth`, `openai` (pointed at OpenRouter), `mongodb`, `@aws-sdk/client-s3` (R2-compatible), `react-dropzone`, `lucide-react`, `clsx`/`tailwind-merge`, `sonner` (toasts).

**Design tokens** ("Continental" — Graphite + Ink Blue), emitted as CSS variables in `globals.css` + a mirrored TS map in `lib/theme/tokens.ts`:
- base graphite `#0E1116`, panel `#161B22`, text `#E6EAF0`, muted `#8B93A7`
- accent ink blue `#3B6EF5`
- risk: critical `#E5484D`, high `#F5A623`, medium `#E8C547`, low `#46A758`
- display **Instrument Serif** (editorial headlines), body **Inter**, data **JetBrains Mono` — loaded via `next/font`
- full token set for radii, shadows, spacing scale; light-mode tokens too, but graphite is the default

**Layout shell**: dense, professional app chrome — top bar with wordmark + nav (Analyze / Compare / Playbooks / History), persistent anonymous-session indicator, keyboard-focus rings everywhere, `prefers-reduced-motion` respected via a Framer `MotionConfig` wrapper.

## Architecture: Ports & Adapters (the key decision)

Three boundaries, each with a single interface + two implementations, chosen at boot by env:

```
src/lib/
  ai/      adapter.ts (interface) · openrouter.ts · mock.ts · pipeline.ts · prompts/{extract,analyze,synthesize}.ts
  storage/ adapter.ts · r2.ts · local.ts
  db/      adapter.ts · mongo.ts · memory.ts
  session/ cookie.ts   # reads/sets `docuintel_session` (UUID), no login
```

- `USE_MOCK=1` (default when no keys) → mock AI (fixture findings with real char-offsets), local filesystem storage, in-memory DB. **Every stage runs and is verifiable now.**
- With `OPENROUTER_API_KEY` / `MONGODB_URI` / R2 creds in `.env.local` → real adapters take over with no component changes.

## 1. Design tokens + core layout shell
- Implement token CSS vars, fonts, base primitives (Button, Card, Badge, Table, Tabs, Dialog, Tooltip, ScrollArea, Separator, Skeleton) customized off the defaults.
- App shell + route skeletons for `/`, `/analyze`, `/compare`, `/playbooks`, `/history`.
- Empty states for each route (premium, not generic).

## 2. Upload + text-extraction pipeline (end to end)
- `POST /api/upload` (multipart, multi-file, drag-and-drop via `react-dropzone`).
- Extractor: `pdf-parse` for PDF, `mammoth.extractRawText` for DOCX, plain passthrough for TXT/MD. Returns normalized text.
- **Visible processing stages** (`extracting → chunking → indexing`) as real state transitions, not a spinner — streamed as SSE progress to the client.
- Persist file to storage adapter, record doc metadata in DB keyed by session.

## 3. Single-pass AI analysis (default rubric)
- Default rubric (`lib/rubric/default.ts`): auto-renewal, liability caps, indemnification, termination, payment terms, IP/warranty, data/confidentiality, governing law — each with severity guidance.
- **Pass 2 (analyze)** against rubric → structured JSON findings: `{category, severity, quote, explanation, suggestedFix, confidence, sectionIndex}`. JSON-schema response format with fenced-JSON fallback extraction.
- Results dashboard v1: risk score (severity-weighted), category breakdown, filterable/sortable findings table. Wired to the mock so it renders real data immediately.

## 4. Signature feature: inline source highlights
- **Pass 1 (extract)**: returns sections + char offsets + doc metadata.
- Findings carry an exact `quote` (verbatim source substring); resolver anchors each finding to its span (offset-first, **substring-fallback** so highlights survive imperfect LLM offsets).
- Document viewer renders the original text with highlighted spans color-coded by severity. Clicking a finding in the table → smooth `scrollIntoView` + pulse-highlight the exact clause (Framer Motion, reduced-motion aware). Hovering a span → popover with the finding. This is the interaction that has to feel exceptional.

## 5. Full multi-pass pipeline + custom playbook
- Wire passes 1→2→3 as separate calls with retry/backoff (rate limits, timeouts) and per-call error isolation.
- **Pass 3 (synthesize)**: streamed executive summary (SSE) + top-3 issues; risk score recomputed deterministically from findings for consistency.
- **Custom playbook**: upload a policy/standard-terms doc → extract into a structured baseline → Pass 2 runs against *baseline + rubric* → diff-style "standard says X / document says Y" per finding. Playbooks stored per session in DB.

## 6. Version comparison
- Upload two versions of one doc → run extraction on both → clause-level alignment (by heading + fuzzy text match) → per-clause diff (added/removed/modified) with a risk-delta verdict (↑/↓/neutral) per changed clause. Dedicated two-pane compare view.

## 7. Feedback loop + session history
- Each finding: "Confirm" / "Not relevant" actions, persisted per session.
- "Not relevant" findings feed forward: similar subsequent findings (same category + similar quote) are auto-deprioritized/hidden-with-rationale in that session.
- `/history`: list of past analyses in this browser session (cookie-id keyed in DB), with re-open + re-export.

## 8. Polish pass
- **PDF export** of the full report (branded, exec summary + risk breakdown + findings table + appendix) via the document-skills PDF route.
- Motion audit: page transitions, result reveal, stage progression — purposeful only.
- Empty states, error states (extraction failed, AI timeout, malformed JSON), rate-limit messaging.
- Mobile responsiveness across all views, focus-visible audit, contrast audit, loading skeletons.

## Cross-cutting
- Strict domain types (`lib/ai/types.ts`: `Finding`, `Section`, `AnalysisResult`, `Playbook`, `VersionDiff`).
- Env contract documented in `.env.example`; OpenRouter model configurable via `OPENROUTER_MODEL` (default a strong long-context structured-output model).
- `README.md` with run instructions for both mock and live modes.
